# Security Policy

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub issues.**

Send reports to: **privacy@skeleton.ai-mds.com** with subject `[SECURITY]`

### What to Include
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

### Response Timeline
- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 7 days
- **Fix timeline**: Within 30 days (critical vulnerabilities prioritized)

### Scope
- Smart contract vulnerabilities
- Encryption/decryption flaws
- Authentication bypasses
- Data leakage
- Access control issues

### Out of Scope
- Social engineering
- DDOS attacks
- Third-party service vulnerabilities (report to the service provider)

## Bug Bounty

Currently community-funded. Report valid critical vulnerabilities and receive credit in our security hall of fame. As funding grows, we plan to offer monetary bounties.

## Security Audit Status

- **Internal review**: ✅ Complete (see docs/SECURITY_ANALYSIS.md)
- **External audit**: 🔜 Planned (seeking community funding)

## Responsible Disclosure

We follow responsible disclosure. We ask that you:
1. Report vulnerabilities privately first
2. Allow us 90 days to fix before public disclosure
3. Do not access or modify other users' data
4. Act in good faith
